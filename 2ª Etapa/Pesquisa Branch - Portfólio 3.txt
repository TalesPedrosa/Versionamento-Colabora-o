Uma Branch é uma "linha de desenvolvimento independente", que permite trabalhar em um conjunto de alterações fora do código principal. Um exemplo para facilitar o entendimento é quando estamos escrevemos um texto ou uma história e queremos testar ou experimentar algo sem estragar o registro oficial ou a história principal, então escrevemos as ideias em outro caderno que seria a nossa "branch". Sendo assim, uma cópia separada onde podemos alterar sem afetar a nossa história principal, que seria o "Main" no GitHub.
Portanto, ela é uma ferramenta essencial para o controle de versão e gerenciamento de projetos no GitHub. O objetivo principal de uma branch e possibilitar que o autor/Dev possa trabalhar em novidades, alterações, testes sem que o arquivo principal, ou seja, o main, seja alterado permanentemente, mas caso a alteração seja agradável, possa integrá-la no projeto principal, ou se não for, seja possível apagar

Aplicações praticas:

1 - Corrigir um erro sem parar o sistema: Um aplicativo que já está em uso por várias pessoas e, aparece um erro no login. Em vez de mexer diretamente no sistema que está funcionando, o Dev cria uma branch para corrigir o problema. Assim, ele testa a solução com segurança antes de colocá-la no sistema original.

2- Criar uma nova função: Um site quer adicionar um botão de "Esqueci minha senha". O programador cria uma branch para desenvolver essa nova função. Enquanto isso, o site continua funcionando normalmente para os usuários. Quando tudo estiver pronto e testado, a nova função é adicionada ao sistema principal.

3 - Várias pessoas trabalhando juntas: Cada pessoa trabalhando em uma parte diferente em um projeto. Por exemplo, uma faz a tela de cadastro e outra cria a tela de login. Cada uma usa sua própria branch, evitando que uma atrapalhe o trabalho da outra. Depois, todas as mudanças são juntadas em um único projeto.
